## 1. 主要フロー

### F-1: 相談者ジャーニー（検索→予約成立）
```
[一般ユーザー] トップ → 占い師を探す → カテゴリ「タロット」+ 相談形式「チャット」で絞り込み
  → 占い師プロフィール閲覧（サービス・記事・評価確認）→ お気に入り登録
  → 予約リクエスト送信（希望日時3候補・相談概要）→ マイページで pending 確認
[占い師] ダッシュボード → 受信リクエスト確認 → 承認（accepted）
[一般ユーザー] 通知受信 → マイページで accepted 確認
検証: 状態が双方で整合（pending→accepted）、通知発火、トランザクション整合
対応: B-3/4/7/8, B-9, REL-2
```

### F-2: コンテンツ→送客ジャーニー（記事作成→公開→SEO→送客）
```
[占い師] 記事作成（Tiptap、カテゴリ/タグ/アイキャッチ/SEO設定）→ 予約投稿（未来日時 scheduled）
[システム/cron] 公開日時到達 → published 自動遷移 → sitemap.xml に反映
[訪問者] `/blog/[slug]` 閲覧 → SEO 検証（title/description/OGP/JSON-LD Article/canonical）
  → 記事から著者占い師プロフィールへ遷移 → 「相談する」CTA → 予約リクエスト導線
検証: scheduled→published 自動公開、SEO メタ出力、著者統合導線、sitemap 公開のみ
対応: C-2-7, C-1-7, C-4, SEO-9/13/16, REL-3
```

### F-3: 初心者マッチングジャーニー
```
[一般ユーザー・初心者] `/match` → 悩み「恋愛」+ 希望「電話」+ 自由記述
  → ルールベース候補提示 → 候補プロフィール → 予約リクエスト
  （候補0件 → 人気占い師の代替提案 → そこから予約）
検証: マッチング結果、0件代替、予約導線接続
対応: B-6, B-7
```

### F-4: 運営ガバナンスジャーニー
```
[運営 ADMIN] ブログ管理 → 占い師投稿記事のステータス変更（公開承認 ON 時 draft→published）
  → 不適切記事 archive → カテゴリ/タグ管理
  → 占い師/ユーザー/サービス管理
検証: ADMIN RBAC、公開承認フロー（OPEN-8 設定）、archive で公開・sitemap から除外
対応: C-3, B 管理, SEC-2
```

---

## 2. 検証観点
- 状態整合性（複数ロール間での予約状態・記事ステータスの一貫性）
- 通知（アプリ内 + メール任意 OPEN-7）
- トランザクション・冪等性（REL-2）
- 権限境界（各ロールが越権操作不可 SEC-2）

## 3. 前提（Phase 1 設定）
- domain/cc_auth 未設定（OPEN-1/2）の間は local スタブ認証で代替。設定確定後 dev で実連携フロー実行。
- ロール別テストアカウント（一般/占い師/運営）を用意。
