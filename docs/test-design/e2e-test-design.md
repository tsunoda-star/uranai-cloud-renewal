## 1. 対象シナリオ

| # | シナリオ | ステップ | 対応 AC |
|---|---------|---------|--------|
| E-1 | トップ表示・主要導線 | `/` 表示 → ヒーロー/統計/カテゴリ/3ステップ/ピックアップ存在 → 各CTA遷移 | A-1 |
| E-2 | 検索→プロフィール→予約リクエスト | `/advisors` 検索/フィルタ → カード → プロフィール → 予約CTA → ログイン → フォーム送信 → 確認 → マイページ pending | B-1/3/4/7 |
| E-3 | フィルタの URL 反映 | フィルタ適用 → URL クエリ反映 → リロードで再現 → 件数表示 | B-3-6/7 |
| E-4 | マッチング相談 | `/match` 入力 → 候補提示 → プロフィール遷移 → 0件時 代替提案 | B-6 |
| E-5 | 占い師: 予約応答 | 占い師ログイン → ダッシュボード → 受信リクエスト → 承認 → 状態 accepted 反映 | B-7-3/4/5, B-9 |
| E-6 | 記事作成→公開→閲覧→SEO | 占い師 → エディタ作成 → 公開 → `/blog/[slug]` 表示 → title/OGP/JSON-LD 出力検証 | C-1/2 |
| E-7 | 予約投稿→自動公開 | 予約投稿(scheduled) → 公開日時到達(ジョブ) → 公開反映 → sitemap 含む | C-2-7, SEO-13 |
| E-8 | ブログ⇔占い師統合 | 記事詳細 → 著者プロフィール遷移 → 「相談する」CTA → 予約導線 | C-4 |
| E-9 | RBAC 境界 | 一般ユーザーが `/dashboard/posts/new` アクセス → 拒否/リダイレクト | C-2-1, SEC-2 |
| E-10 | 空状態 | 0件カテゴリ/検索 → EmptyState 表示 | B-1-5 |
| E-11 | レスポンシブ | 375/768/1280px で崩れ・横スクロールなし、ハンバーガー動作 | A-1-6, RWD-3 |
| E-12 | アクセシビリティ | キーボード操作、フォーカス可視、aria、axe スキャン 0 critical | A11Y-* |

## 2. 計測・成果物
- ブラウザコンソール/ネットワーク収集、スクリーンショット、録画（`.test-logs/`）。
- Lighthouse（トップ/一覧/プロフィール/記事詳細、モバイル+デスクトップ）: Performance 90+/A11y 100/BP 100/SEO 100。
- axe-core でアクセシビリティ自動検査。

## 3. 認証
- CC-Auth は dev 連携 or テストアカウント（ロール別: 一般/占い師/運営）でログインフロー実行。
- domain/cc_auth 未設定（OPEN-1/2）の間は local スタブ認証で代替し、設定確定後に実連携 E2E を追加。
